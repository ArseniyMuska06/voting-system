\# Voting System — Deployment \& Usage



\## Опис

Даний репозиторій містить веб-застосунок системи голосування, розроблений з використанням фреймворку \*\*Django\*\*. Система підтримує розділення ролей користувачів, створення та проходження опитувань, а також зберігання голосів у нереляційній базі даних з метою підвищення рівня анонімності.



---



\## Вимоги до середовища

Для розгортання та коректної роботи веб-застосунку необхідно мати:



\- сучасний веб-браузер;

\- \*\*Python\*\* версії 3.10 або вище;

\- \*\*MongoDB\*\* (локально або віддалено);

\- менеджер пакетів `pip`.



Усі Python-залежності описані у файлі `requirements.txt`.



---



\## Налаштування та розгортання



Після клонування репозиторію необхідно активувати віртуальне середовище:



&nbsp;   .venv/scripts/activate



Далі слід перейти до каталогу репозиторію `voting\_system` та встановити всі необхідні залежності за допомогою менеджера пакетів `pip`:



&nbsp;   cd voting\_system

&nbsp;   pip install -r requirements.txt



Після встановлення залежностей необхідно виконати міграції фреймворку Django:



&nbsp;   python manage.py makemigrations

&nbsp;   python manage.py migrate



У результаті буде автоматично створено таблиці реляційної бази даних для зберігання інформації про користувачів, опитування та варіанти відповідей відповідно до описаних моделей.



---



\## Робота з MongoDB



Для зберігання голосів використовується нереляційна база даних \*\*MongoDB\*\*.  

Такий підхід дозволяє відокремити інформацію про голосування від персональних даних користувачів та підвищити рівень анонімності системи.



Після встановлення MongoDB веб-застосунок автоматично підключається до бази даних та створює окрему колекцію для зберігання голосів користувачів.



---



\## Запуск застосунку



Після завершення налаштувань баз даних веб-застосунок запускається на локальному сервері командою:



&nbsp;   python manage.py runserver



Після запуску система стає доступною через браузер за локальною адресою, що виводиться у консолі.



---



\## Адміністрування системи



Для керування системою необхідно мати обліковий запис адміністратора.  

Створення адміністратора виконується командою:



&nbsp;   python manage.py createsuperuser



Після виконання команди необхідно ввести ім’я користувача, адресу електронної пошти та пароль. Після авторизації адміністратор отримує доступ до адміністративної панелі керування опитуваннями.
